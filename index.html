需要注意的问题

如果是用 brew 安装 的 mysql 有个地方需要调整下:

  1. INCLUDEPATH 大多数的教程写的都是 <mysql安装路径>/include

    比如：D:/mysql5.7/include 或者 /usr/local/mysql 这样的，但是 brew 安装的mysql 它的include目录下面还有一层目录

    名字叫作 mysql

    所以在配置的时候需要配置成 <mysql安装路径>/include/mysql 这个要特别注意，否则make的时候会提示

    mysql_xxxx.h not found 之类的错误。

    所以在这个路径中，手动添加了红色的 /mysql 之后才得以报错消失：

    INCLUDEPATH += “/usr/local/Cellar/mysql@5.7/5.7.32/include/mysql“

  2. LIBS 大多数的教程写的是 <mysql安装路径>/lib，但是你如果配置成这样子的话，对方会拒绝你的make 并且给你丢了一个异常。有很长的一串，只截图了之后的。关键是红色的文字。

    其实我刚看到也是挺懵的，去找 make[2]: *** [../plugins/sqldrivers/libqsqlmysql.dylib] Error 1 这个文件的时候发现，sqldrivers 目录下根本没有这个文件。于是就把目标放到了 红色框框中的一行。关键是看到了那个linker ，突然想到 <mysql安装路径>/lib 目录下有个软连接，不知道跟这个有没有关系，就是下面图上这个。红色框中的是的链接文件，蓝色框中的是这个链接文件的原身，于是试着改了一下 LIBS 的配置：

    原本的配置

    LIBS += “/usr/local/Cellar/mysql@5.7/5.7.32/lib”

    改成了

    LIBS += “/usr/local/Cellar/mysql@5.7/5.7.32/lib/libmysqlclient.20.dylib“

    于是乎 ，LIBS 路径就变成了下面这样子，添加了红色的 /libmysqlclient.20.dylib

    LIBS += “/usr/local/Cellar/mysql@5.7/5.7.32/lib/libmysqlclient.20.dylib“

最后

上面也说了，遇到的问题是用 brew 安装的mysql 才会有这种问题。如果是手动安装的不清楚是否也是有这种问题。我没有测试。自己在操作的时候如果没有遇到上述情况，就不需要加这边的 /mysql 和 /libmysqlclient.20.dylib

INCLUDEPATH += “/usr/local/Cellar/mysql@5.7/5.7.32/include/mysql“

LIBS += “/usr/local/Cellar/mysql@5.7/5.7.32/lib/libmysqlclient.20.dylib“

处理包含扩展名的文件时：直接拿列中配置的值（如下图的 icon_1 , icon_2 , icon_3 , ...）去规则目录 /Users/obiscr/Workspace/tmp/excel_analysis/foo 下查找，不存在则输出日志。

3.2.2 只配置了文件名

语法：<path>|(suffix)

这种情况下：只配置了文件名，没有扩展名。则需要手动指定可能的扩展名。程序会自动拼接 文件名+扩展名， 然后去规则目录 /Users/obiscr/Workspace/tmp/excel_analysis/foo 下查找，不存在则输出日志。

相比于之前的配置增加了 |(png) 。

/Users/obiscr/Workspace/tmp/excel_analysis/foo

/Users/obiscr/Workspace/tmp/excel_analysis/foo|(png)

3.2.3 只配置了文件名，并且有多个文件扩展名

语法：<path>|(suffix1,suffix2…)

如果有多个扩展名，每个扩展名用 , 隔开。

相比于之前的配置增加了一个新的扩展名 ,jpg 。

/Users/obiscr/Workspace/tmp/excel_analysis/foo|(png)

/Users/obiscr/Workspace/tmp/excel_analysis/foo|(png,jpg)

3.2.4 有后缀名

语法：<path>|(suffix)|{[extension1,extension2…]}

比如有图片 iconName_a.jpg, iconName_b.jpg, iconName_c.jpg ，配置如下：

在扩展名后面添加后缀名|{[suffixes…]}

/Users/obiscr/Workspace/tmp/excel_analysis/foo|(png)

/Users/obiscr/Workspace/tmp/excel_analysis/foo|(png)|{[a,b,c]}

如果存在多个扩展名，并且多个扩展名都有后缀名。则配置如下：

语法： <path>|(suffix1,suffix2…)|{[suffix1_extension1,suffix1_extension2…],[suffix2_extension1,suffix2_extension2…]}

/Users/obiscr/Workspace/tmp/excel_analysis/foo|(png,jpg)

/Users/obiscr/Workspace/tmp/excel_analysis/foo|(png,jpg)|{[a,b,c],[a,d]}

但是有一种情况需要注意：假如说有多个扩展名：png，svg，png 有 a，b，c的后缀，svg没有后缀。配置如下：

语法： <path>|(suffix1,suffix2…)|{[suffix1_extension1,suffix1_extension2…],[]}

需要注意的就是，尽管svg格式的图片是没有后缀的，但是后缀名的占位符 [] 依然需要保留。 下面列举几种正确与错误的配置：

/Users/obiscr/Workspace/tmp/excel_analysis/foo|(png)|{[a,b,c]}
/Users/obiscr/Workspace/tmp/excel_analysis/foo|(png,svg)|{[a,b,c],[a,d]}
/Users/obiscr/Workspace/tmp/excel_analysis/foo|(png,svg)|{[a,b,c]} 有两个扩展名，但是只配置了一组后缀
/Users/obiscr/Workspace/tmp/excel_analysis/foo|(png)|{[a,b,c],[a,d]} 有一个扩展名，但是配置了两组后缀

IDEA Plugin - ExcelReader

0x01 介绍

ExcelReader 是一个解析Excel类格式文件的IDE插件，支持 .xls, .xlsx, .csv 三种格式。大家都知道，在 JetBrains 家的产品中， .xls, .xlsx 两种格式采用的是系统默认的文件打开方式（相对比较麻烦），IDE没有对打开方式做拦截，而 .csv 则是采用文本编辑器打开。因此为了提升效率，需要对这三种文件打开方式进行拦截，让其采用自定义的方式打开，ExcelReader主要就是做这个事情。

0x02 安装

在IDE里面进行安装

打开 “File” - “Settings” - “Plugins”，选择 “Marketplace”，输入 ExcelReader ，点击 install 即可安装到IDE。

不同版本或者平台的IDE，上述的打开方式可能略有差别

通过web安装

点击前往ExcelReader插件主页

0x03 使用

通过ExcelReader打开文件

选中一个后缀是 .xls 或 .xlsx 或 .csv 的文件，鼠标右击选择最上面 Open As ExcelReader 即可打开。

ExcelReader 同时支持快捷键打开，选中文件后，按下对应的快捷键即可打开。不同的平台快捷键略有差异。

Windows平台

  • Ctrl Shift D

MacOS平台

  • Command Shift D

内容过滤

打开一个文件之后，ExcelReader界面最上面可以进行简单的搜索操作。

选择 All Column，是在所有列中搜索；选择某个具体的列，则只在选择的列中进行搜索。

IDEA Plugin - ExcelDiffer

0x01 介绍

ExcelDiffer 是一个对比Excel类格式文件的IDE插件，支持 .xls, .xlsx, .csv 三种格式。

0x02 安装

在IDE里面进行安装

打开 “File” - “Settings” - “Plugins”，选择 “Marketplace”，输入 ExcelDiffer ，点击 install 即可安装到IDE。

不同版本或者平台的IDE，上述的打开方式可能略有差别

通过web安装

点击前往ExcelDiffer插件主页

0x03 使用

鼠标点击选中一个后缀是 .xls 或 .xlsx 或 .csv 的文件，然后右击选择 Add to ExcelDiffer -> Left。然后选择另一个文件，右击选择 Add to ExcelDiffer -> Right 即可自动加载 文件内容，并显示比较结果。

右上角显示的是比较结果（有多少个不相同的地方）。

左上角是操作区域，可以在不同的结果之间跳转。

  • 跳转到上一个不同之处
  • 跳转到下一个不同之处
  • 刷新，从磁盘重新加载当前的文件

界面的最下方显示左右两边选中的同一行的纵向对比。

插入排序

描述

“插入排序”是数列排序的算法之一。

规则

首先让最左端的数字已完成排序，然后取出那些尚未操作的左端的数字，将其与已经操作的左侧的数字比较，如果左边的数字大，交换两个数字。

重复此操作，直到出现一个较小的数字或者数字到达左端。
